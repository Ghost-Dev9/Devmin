Einführung in NFS

NFS ist das standardisierte Dateifreigabeprotokoll, das von Unix-Systemen verwendet wird. NFS ermöglicht es einem System, ein Verzeichnis über das Netzwerk auf ein oder mehrere andere Hosts zu exportieren, sodass Benutzer und Programme auf diesen Hosts auf exportierte Dateien zugreifen können, als wären sie lokal.

Ein NFS-Server ist ein System, das ein oder mehrere Verzeichnisse exportiert, während ein NFS-Client ein System ist, das ein oder mehrere Verzeichnisse von einem Server mountet. Ein Host kann sowohl NFS-Server als auch Client anderer Server sein.

Ein Server steuert, welche Clients ein exportiertes Verzeichnis mounten können, indem er die IP-Adresse des Clients mit einer Liste der erlaubten Hosts für das angeforderte Verzeichnis vergleicht. Der Server kann auch ein Exportverzeichnis als nur-lesbar oder nur-lesbar für bestimmte Clients festlegen.

Im Gegensatz zu anderen Netzwerkdateisystemen muss sich ein NFS-Client nicht beim Server anmelden, wenn er ein exportiertes Verzeichnis mountet. Der Server vertraut dem Client, die Benutzer zu authentifizieren und die ID des aktuellen Benutzers beim Zugriff auf exportierte Dateien bereitzustellen. Daher sollten Sie nur vertrauenswürdige Client-Hosts zulassen, die exportierte Verzeichnisse mounten.
