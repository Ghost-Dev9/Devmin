Asynchrone Schreibvorgänge
"Ja" - Schreibvorgänge auf dem Server werden asynchron durchgeführt. Wenn Ihr NFS-Client eine Schreibanforderung an einen NFS-Server sendet, gibt der Server eine positive Antwort an den Client zurück, bevor er Daten auf seiner Festplatte schreibt. Ihr Client kann dann eine weitere Schreibanforderung senden, ohne auf den Abschluss des vorherigen Schreibvorgangs durch den Server zu warten. Die Schreibleistung ist bei asynchronen Schreibvorgängen besser als bei synchronen. Wenn jedoch der Server nach dem Senden der positiven Antwort an den Client, aber vor dem Schreiben der Daten auf die Festplatte abstürzt, gehen die Daten verloren, und der Client wird nicht benachrichtigt.

"Nein" - Schreibvorgänge auf dem Server werden synchron durchgeführt. Wenn Ihr NFS-Client eine Schreibanforderung an einen NFS-Server sendet, schreibt der Server die Daten auf seine Festplatte, bevor er eine positive Antwort an den Client zurücksendet. Ihr Client muss warten, bis der Server die Daten auf die Festplatte geschrieben hat, bevor er eine weitere Schreibanforderung senden kann. Die Schreibleistung ist bei synchronen Schreibvorgängen nicht so gut wie bei asynchronen.

Der Standardwert ist "Nein".
