Diese Seite zeigt Optionen für entweder einen einzelnen Host, Hosts, deren Namen einem bestimmten Muster entsprechen, oder alle Hosts. Einige der weniger offensichtlichen Optionen sind:
Alsr Benutzer einloggen
Normalerweise, wenn kein Benutzername in der ssh-Befehlszeile angegeben wird, wird der Name des aktuellen Benutzers verwendet, um sich beim Remote-SSH-Server anzumelden. Diese Option kann jedoch verwendet werden, um einen anderen Standardbenutzernamen für einen bestimmten Host oder Hosts anzugeben.

Escape-Zeichen
Beim interaktiven SSH-Login kann das Escape-Zeichen verwendet werden, um die Verbindung zu unterbrechen und sie zu schließen oder zu pausieren.

SSH-Verkehr komprimieren?
Wenn diese Option aktiviert ist, wird der SSH-Client alle Daten, die an diesen Host gesendet werden, mit dem gzip-Algorithmus komprimieren. Dies kann nützlich sein, wenn große Dateien mit scp über eine langsame Verbindung kopiert werden.

Lokale Ports an den Server weiterleiten
In dieser Tabelle können lokale Portnummern eingegeben werden, die vom SSH-Server an einen Host und Port weitergeleitet werden. Dies kann nützlich sein, wenn der einzige Zugang zu einem Netzwerk über SSH-Login zu einer Maschine in diesem Netzwerk erfolgt und andere Dienste wie Web- oder POP-Server zugänglich gemacht werden sollen.

Server-Ports an den lokalen Rechner weiterleiten
In dieser Tabelle können Portnummern auf dem Server eingegeben werden, die an einen Host und Port im Netzwerk des Client-Rechners weitergeleitet werden.
