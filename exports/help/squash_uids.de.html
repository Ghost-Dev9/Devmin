UIDs nicht vertrauen
Dies ist eine spezifische Option für NFSv2.
Der Mapping-Daemon ugidd muss ausgeführt werden.
Zusätzlich zur Remote-Benutzer vertrauen-Sektion ermöglicht diese Option das Festlegen einer Liste von Client-UIDs, die als nicht vertrauenswürdiger Benutzer behandelt werden. Es muss eine durch Kommas getrennte Liste von UIDs oder UID-Bereichen eingegeben werden, z. B. 1,10,20-25,100-150.

exportfs-Optionen: squash_uids, map_daemon
