Erlaubte Hosts
Auf dieser Seite können Sie steuern, welche Hosts auf Ihren Datenbankserver zugreifen dürfen und welche Art der Authentifizierung sie bereitstellen müssen. Jede Zugriffskontrollregel kann auf einen einzelnen Host, ein ganzes Netzwerk, alle Netzwerkhosts oder nur auf Benutzer angewendet werden, die von diesem Rechner aus verbinden. Zusätzlich kann jede Regel entweder auf eine ausgewählte Datenbank oder auf alle Datenbanken auf dem Server angewendet werden.

Bei der Auswahl des Authentifizierungsmodus für erlaubte Hosts stehen die folgenden Optionen zur Verfügung:

Klartext-Passwort
Der Benutzer muss ein Passwort angeben, das mit dem auf der Seite PostgreSQL Benutzer festgelegten übereinstimmt.

Verschlüsseltes Passwort
Der Benutzer muss ein Passwort angeben, das, wenn es verschlüsselt wird, mit dem auf der Seite PostgreSQL Benutzer festgelegten übereinstimmt.

Keine Authentifizierung erforderlich
Es muss kein Passwort angegeben werden.

Verbindung ablehnen
Verbindungen von diesem Host werden überhaupt nicht zugelassen. Dies ist nützlich, um den Zugriff von bestimmten Hosts zu verweigern, wenn Sie einem gesamten Netzwerk Zugang gewährt haben.
