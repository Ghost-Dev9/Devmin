SASL-Sicherheitsoptionen "."
smtp_sasl_security_options
Dieser Parameter gibt die Sicherheitsoptionen an, die der Postfix SMTP-Client für die SASL-Authentifizierung verwendet, wenn er eine Verbindung zu einem Remote-SMTP-Server herstellt.

Die Liste ist ein durch Kommas getrennter Parameter mit einem oder mehreren der folgenden Elemente:

  • noplaintext : Methoden, die Klartext-Passwörter verwenden, nicht zulassen
  • noactive : Methoden, die einem aktiven (nicht wörterbuchbasierten) Angriff ausgesetzt sind, nicht zulassen
  • nodictionary : Methoden, die passiven (Wörterbuch-)Angriffen ausgesetzt sind, nicht zulassen
  • noanonymous : Methoden, die eine anonyme Authentifizierung zulassen, nicht zulassen
  • mutual_auth : Erlaube nur Methoden, die eine gegenseitige Authentifizierung ermöglichen (nicht verfügbar mit SASL Version 1)
