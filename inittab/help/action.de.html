Aktion
Beschreibt, welche Aktion durchgeführt werden soll.

Gültige Aktionen für dieses Feld sind:

Respawn
Der Prozess wird immer neu gestartet, wenn er beendet wird (z. B. getty).

Warten
Der Prozess wird einmal gestartet, wenn das angegebene Runlevel erreicht wird, und init wartet auf seine Beendigung.

Einmalig bei Level
Der Prozess wird einmal ausgeführt, wenn das angegebene Runlevel erreicht wird.

Bootzeit
Der Prozess wird während des Systemstarts ausgeführt. Das Runlevel-Feld wird ignoriert.

Bootzeit, warten bis zur Beendigung
Der Prozess wird während des Systemstarts ausgeführt, während init auf seine Beendigung wartet (z. B. /etc/rc). Das Runlevel-Feld wird ignoriert.

Do nothing
Dies tut nichts.

Wann immer in A, B, C
Ein Prozess, der mit einem ondemand-Runlevel markiert ist, wird immer ausgeführt, wenn das angegebene ondemand-Runlevel aufgerufen wird. Ein Runlevel-Wechsel wird jedoch nicht durchgeführt (ondemand-Runlevels sind `a', `b' und `c').

Beim Standardlevel
Ein initdefault-Eintrag gibt das Runlevel an, das nach dem Systemstart erreicht werden soll. Falls keiner vorhanden ist, fragt init auf der Konsole nach einem Runlevel. Das Prozessfeld wird ignoriert.

Während des Systemstarts
Der Prozess wird während des Systemstarts ausgeführt. Er wird vor allen Bootzeit- oder Bootzeit, warten bis zur Beendigung-Einträgen ausgeführt. Das Runlevel-Feld wird ignoriert.

Strom aus
Der Prozess wird ausgeführt, wenn der Strom ausfällt. Init wird normalerweise darüber informiert, wenn ein Prozess mit einem an den Computer angeschlossenen UPS spricht. Init wartet auf die Beendigung des Prozesses, bevor es fortfährt.

Strom aus ohne Warten
Wie bei Strom aus, jedoch wartet init nicht auf den Abschluss des Prozesses.

Strom wiederhergestellt
Dieser Prozess wird ausgeführt, sobald init informiert wird, dass der Strom wiederhergestellt wurde.

Stromausfall
Dieser Prozess wird ausgeführt, wenn init mitgeteilt wird, dass die Batterie des externen UPS fast leer ist und der Strom ausfällt (vorausgesetzt, das externe UPS und der Überwachungsprozess können diesen Zustand erkennen).

Ctrl-Alt-Del
Der Prozess wird ausgeführt, wenn init das SIGINT-Signal erhält. Dies bedeutet, dass jemand auf der Systemkonsole die Tastenkombination CTRL-ALT-DEL gedrückt hat. In der Regel möchte man eine Art shutdown ausführen, um entweder in den Einzelbenutzermodus zu wechseln oder die Maschine neu zu starten.

Besondere Tastenkombination
Der Prozess wird ausgeführt, wenn init ein Signal vom Tastaturhandler erhält, dass eine besondere Tastenkombination auf der Konsolen-Tastatur gedrückt wurde.
