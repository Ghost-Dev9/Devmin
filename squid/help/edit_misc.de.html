Sonstige Optionen
Startup-DNS-Testadressen: DNS-Server, die Squid abfragt, um zu testen, ob DNS-Lookups ordnungsgemäß funktionieren.

SIGUSR1-Logfile-Rotationen: Standardwert ist 10 oder 0, abhängig vom System. Gibt die Anzahl der Logfile-Rotationen an, die bei 'squid -k rotate' durchgeführt werden. Standard ist 10 (Rotation mit Erweiterungen 0–9). Ein Wert von 0 deaktiviert die Rotation, aber die Logdateien werden weiterhin geschlossen und neu geöffnet, sodass sie manuell umbenannt werden können, bevor das Rotationssignal gesendet wird.

Hinweis: Der Befehl 'squid -k rotate' sendet normalerweise ein USR1-Signal an den laufenden Squid-Prozess. In bestimmten Situationen (z. B. unter Linux mit Async-I/O) wird USR1 für andere Zwecke verwendet. Deshalb sollte 'squid -k rotate' anstelle von 'kill -USR1 ' verwendet werden.

Fehlermeldungstext: HTML-Text für Fehlermeldungen. Kann als 'mailto'-URL zur Administratoradresse oder als Link zur Website der Organisation genutzt werden. Um dies in Fehlermeldungen einzufügen, müssen die Fehler-Template-Dateien im 'errors'-Verzeichnis angepasst werden. Einfügen der Variable '%L' im Template zeigt den 'err_html_text'.

Pro-Client-Statistiken: Falls die Sammlung von Client-Statistiken deaktiviert werden soll, muss 'client_db' abgeschaltet werden.

X-Forwarded-For-Header: Standardwert ist 'on'. Falls aktiviert, fügt Squid die IP-Adresse oder den Hostnamen des Systems in die weitergeleiteten HTTP-Anfragen ein. Standardmäßig sieht dies so aus:

X-Forwarded-For: 192.1.2.3

Falls deaktiviert, wird stattdessen ausgegeben:

X-Forwarded-For: unknown

ICP-Anfragen loggen: Standardwert ist 'on'. Falls aktiviert, werden ICP-Anfragen in 'access.log' protokolliert. Falls die ICP-Last sehr hoch ist, kann diese Option deaktiviert werden, um die Performance zu verbessern oder die Log-Analyse zu vereinfachen.

Minimale direkte Hops: Standardwert ist 4. Falls ICMP-Pinging aktiviert ist, werden direkte Abrufe für Seiten durchgeführt, die nicht mehr als diese Anzahl an Hops entfernt sind.

Speicher für zukünftige Nutzung behalten: Standardwert ist 'yes'. Falls aktiviert, behält Squid bereits zugewiesene, aber ungenutzte Speicherpools für zukünftige Nutzung bei. Falls das System wenig Speicher hat und die malloc-Bibliothek des Systems effizienter ist als Squid, sollte diese Option deaktiviert werden.

Beizubehaltender Speicher: Standardwert ist 'Unlimited'. Falls ein Wert größer als 0 gesetzt wird, behält Squid höchstens die angegebene Menge an zugewiesenem (aber ungenutztem) Speicher in Memory-Pools. Speicher, der dieses Limit überschreitet, wird vom malloc-System des Betriebssystems verwaltet. Squid reserviert keinen Speicher im Voraus, sondern speichert nur Objekte, die sonst freigegeben würden. Daher kann dieser Wert relativ hoch angesetzt werden.

Durchzuleitende Header: Standard ist 'Alle Header'. Ersetzt die veraltete 'http_anonymizer'-Option durch eine flexiblere Steuerung. Hier können explizit die Header angegeben werden, die erlaubt oder entfernt werden sollen.

Diese Option kann auf zwei Arten genutzt werden: Entweder durch Zulassen bestimmter Header (wodurch alle anderen entfernt werden) oder durch Entfernen bestimmter Header (wodurch alle anderen erhalten bleiben).

Gefälschter User-Agent: Standardwert ist 'none'. Falls der 'User-Agent'-Header mit der Option Durchzuleitende Header gefiltert wird, lehnen einige Webserver die Anfrage ab. Mit dieser Option kann ein alternativer User-Agent gesetzt werden.

WCCP-Router-Adresse: Definiert den WCCP-„Home“-Router für Squid.

WCCP-Eingangsadresse: Falls erforderlich, kann Squid so konfiguriert werden, dass WCCP-Nachrichten nur über eine bestimmte Netzwerkschnittstelle empfangen werden. Falls unklar ist, wie viele Schnittstellen vorhanden sind, oder nur eine genutzt wird, sollte diese Option nicht verwendet werden.

WCCP-Ausgangsadresse: Falls erforderlich, kann Squid so konfiguriert werden, dass WCCP-Nachrichten nur über eine bestimmte Netzwerkschnittstelle gesendet werden. Falls unklar ist, wie viele Schnittstellen vorhanden sind, oder nur eine genutzt wird, sollte diese Option nicht verwendet werden.
